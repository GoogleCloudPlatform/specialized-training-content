## 0.8 Confirm the deployed MCP server

24. Click **Navigation menu** > **Cloud Run**.

25. Click the **gcs-mcp-server** service.

26. Click the **Security** tab and note the **Authentication** setting. Because this MCP server exposes Cloud Storage operations, it requires authentication to prevent unauthorized access. You will need to supply an identity token when connecting.

27. Open Cloud Shell.

28. In Cloud Shell, run the following command to start the **MCP Inspector** utility:

    ```bash
    npx @modelcontextprotocol/inspector
    ```

    Press ENTER when prompted with `OK to proceed?`

    > **Note:** You will use the Cloud Run proxy to allow the MCP Inspector to connect to the Cloud Run service using your user credentials.

29. Return to the Cloud Shell browser tab and click the **+** icon to open a second terminal session.

30. Start the Cloud Run proxy with the following command:

    ```bash
    gcloud run services proxy gcs-mcp-server \
      --region=us-central1 \
      --port=8081
    ```

    > **Note:** Because you are working in the Cloud Shell environment, you need to do a little extra work to make everything accessible. You will create a proxy server that serves both the Inspector and the Cloud Run proxy on the same port, with different URL paths routing to the different endpoints.

31. Return to the Cloud Shell browser tab and click the **+** icon to open a third terminal session.

32. Create the proxy script by running the following command:

    ```bash
    cat > ~/mcp_proxy.js << 'EOF'
    const http = require('http');

    const INSPECTOR_PORT = 6274;
    const PROXY_PORT = 8081;
    const LISTEN_PORT = 9000;

    const server = http.createServer((req, res) => {
        // Determine if this goes to MCP Server (8081) or Inspector UI (6274)
        const isApi = req.url.startsWith('/mcp-api');
        const targetPort = isApi ? PROXY_PORT : INSPECTOR_PORT;
        const targetPath = isApi ? req.url.replace('/mcp-api', '') || '/' : req.url;

        const proxyReq = http.request({
            host: '127.0.0.1',
            port: targetPort,
            path: targetPath,
            method: req.method,
            headers: { ...req.headers, host: `localhost:${targetPort}` }
        }, (proxyRes) => {
            // Add CORS headers to every response
            res.setHeader('Access-Control-Allow-Origin', '*');
            res.setHeader('Access-Control-Allow-Methods', 'GET, POST, OPTIONS');
            res.setHeader('Access-Control-Allow-Headers', '*');

            // Forward status and headers
            res.writeHead(proxyRes.statusCode, proxyRes.headers);

            // Pipe the data in real-time (No buffering)
            proxyRes.pipe(res);
        });

        proxyReq.on('error', (e) => {
            console.error(`Proxy error on ${targetPort}:`, e.message);
            res.writeHead(502);
            res.end('Backend unreachable');
        });

        // Handle incoming POST data (crucial for tools/list)
        req.pipe(proxyReq);
    });

    // Handle CORS Preflight (OPTIONS)
    server.on('upgrade', (req, socket, head) => socket.destroy()); // No websockets needed

    server.listen(LISTEN_PORT, '0.0.0.0', () => {
        console.log(`MCP Gateway running on port ${LISTEN_PORT}`);
    });
    EOF
    ```

33. Start the proxy:

    ```bash
    node ~/mcp_proxy.js
    ```

34. In Cloud Shell, click **Web preview** > **Change port** in the upper-right corner.

35. Set the **Preview Port** to `9000` and click **Change and Preview**. A new browser tab opens with the MCP Inspector interface.

36. In **MCP Inspector**, set the transport type to **Streamable HTTP**.

37. Copy the URL from the address bar of the browser and paste it into the URL field in **MCP Inspector**.

38. Modify the URL by adding `/mcp-api/mcp` to the end.

39. Set the **Connection Type** to **Direct**.

40. Click **Connect**.

41. In the MCP Inspector, click **Tools** and verify that the following four tools are available:

    | Tool                         | Description                                  |
    | ---------------------------- | -------------------------------------------- |
    | list_objects                 | Lists objects in a Cloud Storage bucket      |
    | read_object                  | Reads text content of a Cloud Storage object |
    | generate_upload_signed_url   | Generates a signed URL for uploading files   |
    | generate_download_signed_url | Generates a signed URL for downloading files |

42. Click the **list_objects** tool, enter `PROJECT_ID-cymbal-meet-refs` as the **bucket** parameter, and click **Run Tool**. Verify that the tool returns the list of reference PDFs stored in the bucket.
