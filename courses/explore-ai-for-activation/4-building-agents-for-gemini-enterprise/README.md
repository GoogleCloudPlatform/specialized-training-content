# Building Agents for Gemini Enterprise

## What students learn
Students learn how to design, build, and operationalize agents in Gemini Enterprise. They start with simple prompt-based agents, then use Agent Designer for configuration, create multi-agent routing patterns, and set up scheduled agents for autonomous recurring workflows.

## Case study
This course uses Cymbal Insurance, a fictional enterprise handling high volumes of claims, policy updates, and customer requests. Learners build agent workflows to improve triage, task specialization, and reliable automation in a complex operations environment. In the final lab, students step outside the scenario entirely — defining and building an agent for a use case of their own choosing.

## Labs

| # | Lab | Description |
|---|-----|-------------|
| 1 | [Creating Simple Agents](1-creating-simple-agents/creating-simple-agents.md) | Create two prompt-based agents: a Claims Briefing Assistant that structures incident reports, and a Claims Escalation Desk that triages and routes claims automatically. |

| 3 | [Creating Multi-Agent Systems](3-creating-multi-agent-systems/creating-multi-agent-systems.md) | Build a New Hire Onboarding Orchestrator that delegates tasks to three specialized sub-agents simultaneously, then extend the system independently in the bonus. |
| 4 | [Scheduling Agents for Autonomous Workflows](4-scheduling-agents-for-autonomous-workflows/scheduling-agents-for-autonomous-workflows.md) | Create a web-connected agent with a daily recurring schedule that autonomously delivers a competitor intelligence brief each morning. |
| 5 | [Bring Your Own Agent Use Case](5-bring-your-own-agent-use-case/bring-your-own-agent-use-case.md) | Apply the full agent-building workflow independently to a use case of your own design. |
