ADK BigQuery Agent

Using the official ADK authentication pattern

Why you're asked to sign in on your first message

This agent's BigQuery capability is a remote, OAuth-protected MCP server. To build that toolset, ADK has to connect to the MCP server and list its tools — and that connection itself requires OAuth. ADK does this at the start of your first request (before the model even decides whether it needs data), so the sign-in prompt appears once, up front, no matter what you ask.

After you authorize, the token is cached for the rest of your session, so you won't be prompted again. The flow:
1. Your first message triggers the MCP connection → ADK emits an auth_required event with an OAuth URL
2. You click "Login with Google" and authorize
3. The client returns the result as a FunctionResponse
4. ADK exchanges and caches the token, then continues
